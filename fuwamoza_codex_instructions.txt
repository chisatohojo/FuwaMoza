# Codex向け指示書: ふわもざ

## 目的

Windows向けの画像加工アプリ「ふわもざ」を作成してください。

「ふわもざ」は、画像内の任意の場所に「モザイク」または「ぼかし」をブラシ感覚で追加し、元画像と同じサイズ・同じ画像形式で保存できる軽量アプリです。

主な用途は、スクリーンショットや写真に写った個人情報、顔、ナンバープレート、不要な文字などを、クリックまたはドラッグ操作で簡単に隠すことです。


## アプリ名

ふわもざ

英字表記が必要な箇所では以下を使用してください。

FuwaMoza


## 技術構成

以下の構成を基本にしてください。

- Python 3.11 以降
- PySide6
- Pillow
- PyInstaller で exe 化できる構成

OpenCV は必須ではありません。
Pillow だけで実装できる場合は Pillow を優先してください。


## 基本要件

1. Windows デスクトップアプリとして動作すること。
2. 画像ファイルをウィンドウへドラッグアンドドロップすると読み込めること。
3. 対応画像形式は PNG, JPG, JPEG, BMP, WEBP とすること。
4. 読み込んだ画像を中央にプレビュー表示すること。
5. 画像の表示サイズがウィンドウに合わせて拡大縮小されること。
6. プレビュー上でクリックまたはドラッグして、効果をかける範囲を指定できること。
7. 効果タイプとして「モザイク」と「ぼかし」を切り替えられること。
8. ブラシの太さをスライダーで変更できること。
9. 効果の強さをスライダーで変更できること。
10. 複数箇所に効果を追加できること。
11. Undo で直前の操作を取り消せること。
12. クリアで読み込み直後の状態に戻せること。
13. 保存ボタンで加工済み画像を保存できること。
14. 保存時、元画像と同じ幅・高さで出力すること。
15. 保存時、元画像と同じ画像形式で出力すること。
16. 元画像は上書きしないこと。
17. 保存ファイル名は、元ファイル名の拡張子前に「_fuwamoza」を追加すること。

例:
- sample.png -> sample_fuwamoza.png
- photo.jpg -> photo_fuwamoza.jpg
- screenshot.webp -> screenshot_fuwamoza.webp

18. 同名ファイルがすでに存在する場合は、連番を付けて保存すること。

例:
- sample_fuwamoza.png
- sample_fuwamoza_2.png
- sample_fuwamoza_3.png

- 画像未読み込み時: 「画像をここにドラッグ＆ドロップ」と表示

## 操作仕様

### 画像読み込み

- ドラッグアンドドロップで画像を読み込めるようにしてください。
- ファイル選択ダイアログからも画像を開けるようにしてください。
- 非対応ファイルが渡された場合は、メッセージボックスでエラーを表示してください。
- 新しい画像を読み込む場合、現在の編集内容が未保存なら確認ダイアログを表示してください。


### クリック操作

プレビュー上をクリックした場合、クリック位置を中心に円形ブラシ範囲へ現在選択中の効果を適用してください。

- ブラシ太さは直径として扱ってください。
- 実画像座標に変換してから処理してください。
- プレビュー表示倍率や余白による座標ズレが起きないようにしてください。


### ドラッグ操作

プレビュー上でマウスを押しながらドラッグした場合、ペンでなぞるように線状に効果を適用してください。

- 線の太さはブラシ太さに従ってください。
- 点と点の間が途切れないよう、前回座標から現在座標まで線を描いてください。
- クリックとドラッグは同じブラシ処理の延長として扱ってください。
- マウスを離したタイミングで1操作としてUndo履歴に登録してください。


## 画像処理仕様

画像処理は、直接範囲を書き換えるだけではなく、可能であれば以下の構造で実装してください。

1. 元画像 original_image を保持する
2. 現在の編集結果 current_image を保持する
3. クリックまたはドラッグ操作ごとに、効果適用前の current_image をUndo履歴に保存する
4. ブラシ操作用にマスク画像を生成する
5. マスク範囲だけに効果画像を合成する

### モザイク処理

モザイク処理は以下の方式にしてください。

1. 対象範囲または画像全体を小さく縮小する
2. 最近傍補間で元サイズに拡大する
3. マスク範囲だけ current_image に合成する

強さの扱い:
- 強さが大きいほどモザイクのブロックを大きくしてください。
- 例: strength = 16 の場合、16px程度のブロック感にする。
- 最低でも 1 で破綻しないようにしてください。

### ぼかし処理

ぼかし処理は Pillow の GaussianBlur を使ってください。

強さの扱い:
- 強さが大きいほどぼかしを強くしてください。
- 例: strength = 8 なら GaussianBlur(radius=8)
- 最大値でも処理が重くなりすぎないように注意してください。


## 座標変換の重要要件

最重要です。

プレビュー上でクリック・ドラッグした座標と、実画像上の座標が必ず一致するようにしてください。

考慮すること:
- 画像プレビューがウィンドウに合わせて縮小表示されている
- アスペクト比維持で表示される
- 画像の左右または上下に余白が出る可能性がある
- その余白部分をクリックしても画像処理しない
- プレビュー座標から実画像座標へ正しく変換する

実装では以下の情報を明確に管理してください。

- 表示中画像の左上座標
- 表示中画像の幅と高さ
- 実画像とのスケール比
- プレビュー上のクリック座標
- 変換後の実画像座標


## 保存仕様

保存ボタンを押したとき、現在の編集結果を保存してください。

要件:
- 元画像と同じ幅・高さで保存する
- 元画像と同じ画像形式で保存する
- 元画像を上書きしない
- 元画像と同じフォルダへ保存する
- ファイル名は拡張子前に「_fuwamoza」を追加する
- 同名ファイルがある場合は連番を付ける
- 保存成功時は保存先パスをメッセージボックスなどで表示する

JPEG保存時:
- 可能なら品質劣化を抑えるため quality=95 程度で保存する
- 透過情報がある形式からJPEGへ変換するケースは基本想定しなくてよい
- 元がJPEGならJPEGとして保存する

PNG保存時:
- 透過がある場合は透過を保持すること

WEBP保存時:
- Pillowで保存できる範囲で対応すること


## Undo / クリア仕様

Undo:
- 直前のクリックまたはドラッグ操作を取り消す
- ドラッグ中の細かい点ごとではなく、マウスを押してから離すまでを1操作として扱う
- Undo履歴は最大20件程度でよい

クリア:
- 読み込み直後の元画像状態に戻す
- Undo履歴もクリアする


## エラー処理

以下の場合はメッセージボックスで分かりやすく通知してください。

- 画像を読み込めない
- 非対応形式
- 保存に失敗
- 画像未読み込みのまま保存を押した
- 予期しない例外が発生した

## READMEに書くこと

README.md には以下を記載してください。

- 実行方法
- 使い方
- exe化方法
- 注意事項

実行方法の例:
```bash
pip install -r requirements.txt
python main.py
```

exe化コマンドの例:
```bash
pyinstaller --noconsole --onefile --name FuwaMoza main.py
```

アイコンを指定する場合:
```bash
pyinstaller --noconsole --onefile --name FuwaMoza --icon assets/icon.ico main.py
```


## 仕上げ要件

- コードには適度にコメントを入れてください。
- ユーザーが迷わないUIにしてください。
- Windowsでの使用を前提に、パス処理は pathlib を使って安全に実装してください。
- 画像処理の座標ズレが起きないよう、特に注意してください。
- まずは確実に動くMVPを優先してください。
- 余計な大型機能は入れず、軽量で使いやすいアプリにしてください。
